Optimizing Your Website By Extracting Unused CSS from HTML Files

The optimization of efficient code is one of the most important roles in a web developer's life. Your website's CSS files can quickly bloat up with unused styles over time, especially if you are using a very popular framework like Bootstrap or working with pre-built templates. These unused styles will ultimately slow down your website and make your code difficult to handle. In this post, we'll learn how to extract and remove unused CSS from your HTML files using tools such as PurgeCSS and UnCSS. We will further look at explicit scenarios, for example, when working with CSS frameworks and ready-built templates, and talk about some online tools at your disposal which let you simply paste your HTML and CSS to quickly optimize them.

The Problem with CSS Frameworks

CSS frameworks like Bootstrap, Tailwind, Foundation, and many more are really good ways of speeding up your development process by quickly giving you responsive, rich-featured websites. All of these frameworks, however, come along with a large volume of CSS that you might never use. Here's an example:

  • Bootstrap: A full-fledged framework with styles for buttons, forms, modals, tooltips, and lots of other elements. If your project uses only a few of these components, though, the rest of the CSS goes to waste, ballooning your file size.

  • Tailwind CSS: Although Tailwind championed a utility-first methodology, encouraging one to use only those classes that one needs, a typical Tailwind installation still comprises thousands of utility classes by default.

Why This is a Problem

  1. Performance Impact: Every style not in use adds to the size of your CSS files, meaning increased download time and, correspondingly, slower page loads.
  2. Complexity: It's difficult to maintain a huge CSS file with lots of unused styles as your project grows.
  3. Overhead: More CSS files give the browser more work to do when rendering your page.

Removing Unused CSS from Frameworks

One may do this using tools like PurgeCSS and UnCSS. Both these tools extract only the used CSS in your project to solve this issue.

Using PurgeCSS with Frameworks

PurgeCSS works really well for projects which use a CSS framework. You can specify which HTML and JavaScript files should be scanned to make sure that only the styles really used in your project are kept.

Example: Setting up PurgeCSS in your build process if you're using Bootstrap: 

var Purgecss = require('purgecss')
    var purgecss = new Purgecss({
      content: ['**/*.html'],
      css: ['**/*.css']
    })
    var purgecssResult = purgecss.purge()

This configuration will look through your HTML and JavaScript files for any Bootstrap classes being used, and remove the rest.

Tailwind CSS Example: Tailwind CSS has native support for PurgeCSS, which provides even easier removal of styles that are not used. Configuration is done in your tailwind.config.js file as follows: 

module.exports = {
      purge: ['./src/**/*.html', './src/**/*.js'],
      // other config options
    };

Using UnCSS and Frameworks

With UnCSS, you can remove unused CSS from frameworks. This solution, though generally easier to set up for small projects, is quite powerful for static site setups using Bootstrap.

Dealing with Pre-Built Templates

Another common scenario is you start with a pre-built HTML/CSS template, and then you further modify it to your taste. You might delete some HTML elements or even sections that you don't need, but the related CSS stays there, bloating your stylesheet for no reason.

The Challenge

  • Template Complexity: Most pre-built templates come with solid CSS that covers a wide range of components and complex layouts. As soon as you begin the process of removing HTML elements, related CSS is left over, unused and possibly perplexing.
  • Manual Cleanup: It's time-consuming and full of errors to remove by hand unused CSS from a template in a big project.

Automated Solutions: In such scenarios, tools such as PurgeCSS and UnCSS really show their worth. They will delete CSS that is no longer in use once you have made changes to your template.

How to Optimize a Customized Template

  1. Parse the HTML Files: Run the tools, PurgeCSS or UnCSS on your HTML files after modification.
  2. Remove Unused CSS: Both of these tools auto-delete any CSS rules no longer applying to the new structure of the HTML.
  3. Test and Refine: Always test your website thoroughly after running these tools to make sure that no essential styles were removed by mistake.

This approach ensures that the CSS used by the remaining HTML elements is retained.

Extracting Unused CSS Using Online Tools

On top of the above mentioned CLI tools, there are quite a lot of online tools to which you can just paste your HTML and CSS to quickly purge unused styles. Particularly, these tools become handy for small projects, quick tests, or where you want a fast, easy way to clean up your CSS without setting up a build process.

Here are a few popular online tools

  1. UnCSS Online

  2. UnCSS Online is a no-brain, easy tool. You paste your HTML and CSS, and it returns cleaned-up CSS. Perfect for small, quick tasks and smaller projects.

  3. PurifyCSS Online

  4. The PurifyCSS Online tool allows you to enter the ulr to the website or simply paste your HTML and CSS directly into a web interface. This way, you have a preview of what CSS it is going to purge without having to wire it up directly in your project. This tool is awesome, especially if you're playing with a few different configurations and testing before integrating it into your project.

  5. CSSNano

  6. Even though it is a CSS minifier, it goes further to decrease the file size by eliminating excessive CSS. You can paste your CSS into their online interface to quickly optimize it.

How to Use These Tools

  1. Paste Your HTML and CSS: Paste your HTML and CSS into text boxes on that tool's site.
  2. Run the Tool: At the click of a button, the tool goes through your HTML and CSS and purges all the unused styles.
  3. Download the Optimized CSS: At the completion of the process, a download button appears, clicking on which you can download a cleaned CSS file to replace your old, heavy stylesheet.

Pros of Online Tools

  • Convenience: Nothing to install or configure. Just paste and go.
  • Fast Feedback: Ideal for trivial changes or if you are short of time.
  • No Configuration: Ideal for non-developers, or those who don't want to set up build tools.

Limitations

  • Not Good for Large Projects: These tools are not so great on large complex projects. In that case, build tools like PurgeCSS will be a better fit.
  • Limited Customization: You may not get to control the process as one would in a locally installed tool.

Best Practices for Using CSS Frameworks and Templates

For your CSS to stay optimized while working with these, the following will be of great help:

  1. Modular Frameworks: This means picking frameworks, if possible, that allow you to import only the components that you really want (for instance, Bootstrap's modular CSS files).
  2. Integrate PurgeCSS or UnCSS Early: Include these tools in the beginning of your build process. This will help keep the CSS optimized since your project will evolve.
  3. Watch Out and Clean Up: Keep an eye on your CSS—especially after big changes to your HTML—for unused styles and remove them.
  4. Be careful with Dynamic Content: Be careful while working with these tools on dynamic content. Make sure styles for elements that may or may not exist within the HTML at any given point—think modals, dropdowns—do not accidentally get scoped and removed.

Conclusion

Extracting unused CSS is a crucial step toward optimizing your website's performance and maintainability, whether you're using a popular CSS framework such as Bootstrap or working with a pre-built template. That is where tools like PurgeCSS and UnCSS come in handy with potent solutions for such a challenge, enabling you to maintain clean and efficient CSS. Online tools provide an easy and fast way to strip unused CSS, thus availing optimization even to small project developers.

Build those tools into your workflow and, by following best practices, you can ensure your site is not only beautiful but performs great too. So go ahead and clean up your CSS today, your users and your future self will thank you!
